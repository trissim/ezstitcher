# Grid Size Mismatch Fix Plan

## Status: Complete for ImageXpress, In Progress for Opera Phenix
## Progress: 50%

## 1. Problem Analysis

There's a mismatch between the grid size used in the data generator and the grid size used in the stitcher:

1. In `tests/integration/test_synthetic_microscopes.py`, the synthetic data generator is configured with a grid size of (4, 4) in the `syn_data_params` dictionary.
2. The HTD file generated by `SyntheticMicroscopyGenerator` correctly includes this grid size (4, 4).
3. However, when the `PlateProcessor` tries to parse the HTD file, it either:
   - Fails to parse the grid size correctly, or
   - Successfully parses the grid size but doesn't use it correctly
4. As a result, the default grid size of (2, 2) is used in the `_generate_positions_ashlar` method of the `Stitcher` class.
5. This causes issues because the stitcher expects 4 files (2x2 grid) but finds 16 files (4x4 grid).

The issue is that the `_generate_positions_ashlar` method in the `Stitcher` class checks if there are enough files for the grid size, but it doesn't check if there are too many files for the grid size and adjust accordingly.

## 2. Proposed Solution

Enhance the `_generate_positions_ashlar` method in the `Stitcher` class to:

1. Check if the number of files matches the expected grid size (grid_size_x * grid_size_y).
2. If there are more files than expected, adjust the grid size to match the number of files.
3. Log a warning when this adjustment happens.

This will ensure that the stitcher can handle cases where the grid size is underestimated.
